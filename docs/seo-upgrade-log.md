## 1. Technical SEO

### 1.1 Canonical tag per page
- **Problem:** programmatic sites risk missing or wrong canonicals, causing duplicate-URL dilution.
- **Fix:** `<link rel="canonical" href="{{ .Permalink }}">` renders on every page.
- **Why:** tells Google the one true URL for each page; critical at 6,000+ pages.
- **Files:** `site/layouts/_default/baseof.html:11`.
- **Status:** Applied.

### 1.2 Canonical host is non-www
- **Problem:** the audit flagged a possible www vs non-www mismatch between the repo and Search Console. CLAUDE.md's permanent-facts section claimed www was canonical, contradicting the repo's own `baseURL`, schema, robots.txt and llms.txt, all of which are non-www.
- **Fix:** confirmed with Gareth that `https://pettransportglobal.com/` (non-www) is canonical and www 301-redirects to it. Corrected the CLAUDE.md URL-format block, which had it backwards.
- **Why:** a split canonical host halves link equity and confuses indexation. Every internal link, schema URL and sitemap entry must agree on one host.
- **Files:** `CLAUDE.md` (URL FORMAT block); `site/hugo.toml` `baseURL` was already correct.
- **Status:** Applied. Note: live-review links posted during the design session used www; they resolved via the redirect but future links must use non-www.

### 1.3 robots.txt generation
- **Problem:** an external audit claimed no custom robots.txt exists.
- **Fix / finding:** false finding. `site/layouts/robots.txt` is a Hugo template that generates a correct robots.txt with per-bot allow rules (OAI-SearchBot, ChatGPT-User, PerplexityBot, Claude-SearchBot, Google-Extended, GPTBot, and `*`) and the correct non-www `Sitemap:` line via `.Site.BaseURL`. `enableRobotsTXT = true` in hugo.toml. A template is better than a static file here because it tracks BaseURL automatically.
- **Files:** `site/layouts/robots.txt`, `site/hugo.toml:10`.
- **Status:** Verified-no-change.

### 1.4 Sitemap + split sitemaps
- **Problem:** a single sitemap for thousands of URLs is hard for Google to process and diagnose.
- **Fix:** `split_sitemap.py` classifies URLs and builds per-section sitemaps plus an index, run in the deploy pipeline. Main `sitemap.xml` generated by Hugo.
- **Files:** `split_sitemap.py`, `site/hugo.toml:29` (`[sitemap]`).
- **Status:** Applied.

### 1.5 GSC verification field blank
- **Problem:** `googleSearchConsole = ""` in hugo.toml (a leftover TODO).
- **Fix / finding:** non-issue. Gareth confirmed Search Console is verified via DNS TXT record, so the HTML-tag field is moot. Left blank intentionally.
- **Files:** `site/hugo.toml:64`.
- **Status:** Verified-no-change.

### 1.6 hreflang
- **Problem:** external audit noted no hreflang.
- **Finding:** not needed. The site is English-only. hreflang is only required for localised language/region variants.
- **Status:** Verified-no-change (revisit only if localised versions are ever built).

### 1.7 Indexation quality sampling
- **Problem:** at 6,000+ programmatic pages the real risk is "Crawled, currently not indexed" from thin or near-duplicate routes.
- **Recommendation:** sample ~100 indexed route pages and compare title uniqueness, first-300-words uniqueness, FAQ uniqueness, template overlap, and GSC index status.
- **Files:** would touch none directly; a QA/audit task over `site/content/routes/`.
- **Status:** Outstanding (doable from the repo without external tools; not yet run).

### 1.8 GSC-driven title/meta tuning
- **Problem:** pages ranking position 4-20 with low CTR need title/meta tuning from real impression data.
- **Recommendation:** monthly, tune the top 50 pages by impressions with position 4-20 and CTR below ~1.5%.
- **Status:** Outstanding and currently blocked. No Google Search Console connector is available in-session (only Semrush). Needs a GSC export or connector before it can start.

## 2. On-page

### 2.1 Route `<title>` bug (custom titles not rendering)
- **Problem:** route pages rendered a generic template title instead of the page's own SEO title.
- **Fix:** title templates corrected so per-page titles render (verified: `Shipping Dogs & Cats from Argentina to Australia | PetTransportGlobal`).
- **Why:** the title tag is the strongest on-page ranking and CTR lever; a generic title on thousands of pages is a duplicate-title problem.
- **Files:** route templates + `site/layouts/_default/baseof.html`.
- **Status:** Applied.

### 2.2 H1 title leak on default single template
- **Problem:** `_default/single.html` printed the full SEO title (with ` | PetTransportGlobal` suffix) as the visible H1.
- **Fix:** H1 now takes only the part before the first ` | ` separator: `{{ $h1 := index (strings.Split .Title " | ") 0 }}`.
- **Why:** the H1 should read as a human heading, not an SEO string; one clean keyword-bearing H1 per page.
- **Files:** `site/layouts/_default/single.html`.
- **Status:** Applied.

### 2.3 Per-page meta description and Open Graph
- **Problem:** duplicate or missing meta descriptions hurt CTR and dedup.
- **Fix:** `<meta name="description">` renders from each page's `.Description`; og:title/description/url/type/image all per-page.
- **Files:** `site/layouts/_default/baseof.html:9,14-18`.
- **Status:** Applied.

### 2.4 Broken internal links (sitewide)
- **Problem:** 549 distinct broken internal link targets across roughly 42,232 instances (header origin nav, dead `/routes/` path segment, dead Cost Calculator footer link, dead hardcoded route links in origin bodies, missing path prefixes on blog links). Root cause included 52 origin files missing an explicit `slug:` field, so Hugo fell back to ugly title-derived slugs.
- **Fix:** added explicit `slug:` to all 52 origin files; added `site.GetPage` existence guards before rendering data-derived airline/breed links; removed the dead footer link and dead body links; corrected `/pet-transport/routes/X/` to `/pet-transport/X/`; created shared `valid-sideways.html` filter partial. Rebuild scan confirmed 0 broken internal links (913,375 hrefs checked in the final build).
- **Why:** broken internal links waste crawl budget, break equity flow, and signal low quality at scale.
- **Files:** `site/content/origins/*.md` (52), `site/layouts/partials/airline-breed-sidebar.html`, `site/layouts/partials/routes/*.html`, `site/layouts/partials/footer.html`, `site/content/routes/*.md` (12), 2 blog posts, new `site/layouts/partials/routes/valid-sideways.html`.
- **Status:** Applied.

## 3. Structured data / schema

### 3.1 JSON-LD double-escaping
- **Problem:** structured data was double-escaped across templates, producing invalid JSON-LD.
- **Fix:** `safeJS` / `jsonify` used correctly; verified clean on a sample route page (Organization, BreadcrumbList, WebPage, HowTo, FAQPage all valid, no double escaping).
- **Files:** route + default templates, `site/layouts/_default/baseof.html`.
- **Status:** Applied.

### 3.2 SearchAction / sitelinks searchbox
- **Problem:** the WebSite schema's SearchAction pointed at a non-existent `/pet-transport/routes/{search_term_string}/` URL.
- **Fix:** replaced with a real endpoint `https://pettransportglobal.com/search/?q={search_term_string}` backed by an actual `/search/` page.
- **Files:** `site/layouts/_default/baseof.html:43-48`, `site/layouts/_default/search.html`.
- **Status:** Applied.

### 3.3 Organization schema TODO placeholders
- **Problem:** the `sameAs` array contained `TODO: Facebook URL` etc, which is invalid JSON-LD.
- **Fix:** placeholder strings removed (an absent `sameAs` is valid; placeholder strings are not).
- **Files:** `site/layouts/_default/baseof.html`.
- **Status:** Applied (verified no `TODO` string remains in the schema block).

### 3.4 FAQ / HowTo / Breadcrumb schema per page
- **Problem:** programmatic pages need valid, page-specific structured data.
- **Fix:** FAQPage schema renders from `faqs:` front matter; BreadcrumbList and WebPage/HowTo render per route; author Person schema present.
- **Files:** route templates, `breeds/single.html`, `_default/methodology.html`.
- **Status:** Applied.

## 4. Content quality, information gain, de-duplication

### 4.1 Route prose de-duplication
- **Problem:** near-duplicate route prose at scale is a scaled-content risk.
- **Fix:** de-duplication via data-keyed template composition (`route-overview.html`, `route-sections.html`, `route-quick-answer.html`) so complexity-bearing routes render from structured data rather than repeated stored prose.
- **Files:** `site/layouts/partials/routes/route-overview.html`, `route-sections.html`, `route-quick-answer.html`.
- **Status:** Applied.

### 4.2 Answer-first capsule on routes
- **Problem:** users and Google want the direct answer up top.
- **Fix:** `route-quick-answer.html` capsule renders near the top of all six live route variants.
- **Files:** `site/layouts/partials/routes/route-quick-answer.html` + the six `route-new-*.html`.
- **Status:** Applied.

### 4.3 Quarantine display logic (YMYL accuracy)
- **Problem:** hero showed wrong quarantine information on some routes (notably Australia), a YMYL-accuracy risk.
- **Fix:** corrected quarantine display logic.
- **Files:** route templates.
- **Status:** Applied.

### 4.4 Jamaica regulatory correction
- **Problem:** roughly 10 existing Jamaica routes still state Jamaica is EU-unlisted and requires a FAVN titre test. Jamaica is actually EU Group 2 listed (Implementing Regulation (EU) 2024/1130), no titre test. Newer Jamaica routes are correct; the old ones are not.
- **Recommendation:** correction pass over the existing Jamaica route files.
- **Files:** `site/content/routes/jamaica-to-*.md` (approx 10).
- **Status:** Outstanding (flagged in MEMORY.md).

### 4.5 Template opening-construction footprint
- **Problem:** many route `overview:` fields open with the same construction ("Moving a pet from X to Y…", "Moving to Y from X…", "Shipping your dog or cat from X to Y…"). Two of three sampled routes opened with "Moving…". This is a recognisable skeleton across pages.
- **Recommendation:** vary the opening sentence construction per page (open on the specific fact, price, authority, or edge case that makes each route different), and rotate section order.
- **Files:** route generation logic + `site/content/routes/*.md`; the new build routine must enforce this at generation time.
- **Status:** Outstanding (new-page guardrail; not retrofitted to existing pages).

### 4.6 Information gain per page
- **Problem:** programmatic pages that add nothing new are what Google penalises at scale.
- **Recommendation:** every page must state at least one specific fact or comparison a competitor page does not (named authority, real cost/currency, real timeframe, concrete process, edge case).
- **Status:** Partial. Routes carry route-specific regulatory data (authority names, timelines, quarantine, titre rules), which is genuine information gain, but there is no explicit generation-time gate requiring a unique fact per page. The new routine must add one.
